  Fluxo de Faturamento Anual (do Salão)

  Objetivo: Exibir o faturamento anual por mês, com detalhes de clientes e destacar o mês de maior faturamento/clientes, limpando
  dados com mais de um ano de idade.

  1. Persistência de Dados (Backend - server.js & Banco de Dados)

   * Ação: Quando um novo registro de faturamento é criado (POST /faturamento), o server.js irá salvar o nome do cliente, valor do
     serviço e um timestamp completo (data e hora exatas da inserção) em uma coluna chamada data_completa na tabela faturamento do
     banco de dados.
       * Detalhe: O formato do timestamp será ISO 8601 (ex: "2026-04-29T10:30:00.000Z").
       * Requisito de DB: A coluna data_completa no Postgres deve ser do tipo TIMESTAMP WITH TIME ZONE.

  2. Limpeza Anual de Dados (Backend - server.js)

   * Lógica: A deleção de dados antigos será acionada uma única vez por ano no servidor.
   * Gatilho: Quando o faturamento.html fizer a primeira requisição GET /faturamento em um novo ano (ex: 1º de janeiro de 2027), o
     server.js fará uma verificação.
   * Processo:
       * O server.js terá uma forma (ex: uma variável em memória) de saber se a limpeza do ano anterior para o ano corrente já foi
         realizada.
       * Se ainda não foi feita:
           * Executará um comando SQL DELETE na tabela faturamento para remover todos os registros cujo ano seja anterior ao ano
             atual.
           * Atualizará a flag interna para indicar que a limpeza já foi feita para o ano corrente.
       * Se já foi feita: Ignora o passo de deleção.
       * Resultado: Os dados de faturamento de anos passados (ex: 2025 quando estamos em 2026) serão permanentemente removidos do
         banco de dados.

  3. Recuperação e Processamento de Dados (Backend - server.js)

   * Ação: Após a verificação/execução da limpeza, a rota GET /faturamento buscará todos os registros de faturamento pertinentes ao
     ano atual do banco de dados.
   * Filtro: A query SQL no server.js incluirá um filtro para garantir que apenas os dados do ano corrente sejam retornados para o
     frontend.
   * Envio: Os dados serão enviados como uma lista (array de objetos) para o faturamento.js no frontend.

  4. Agrupamento e Análise de Dados (Frontend - faturamento.js)

   * Ação: O faturamento.js receberá a lista de registros de faturamento do ano atual.
   * Lógica "Caixinhas":
       * Irá inicializar um objeto vazio (as "caixinhas").
       * Percorrerá a lista de registros, e para cada registro:
           * Extrairá o mês do timestamp (data_completa).
           * Adicionará o valor do serviço ao total da "caixinha" correspondente àquele mês.
           * Incrementará a contagem de clientes daquela "caixinha" para o mês.
   * Análise: Após o loop, o faturamento.js terá um objeto com os totais e contagens de clientes por mês. Com base nisso, irá:
       * Identificar qual mês teve o maior faturamento.
       * Identificar qual mês teve o maior número de clientes.
   * Ação: O faturamento.js usará os dados agrupados e analisados para:
       * Exibir uma lista de cada mês com seu total de faturamento e a lista de clientes.
       * Mostrar claramente qual mês faturou mais e qual teve mais clientes.
   * Design: A apresentação seguirá as convenções de UI/UX do restante do site.

   Tabelas: 

      CREATE TABLE IF NOT EXISTS servicos (
      id SERIAL PRIMARY KEY,
      nome TEXT,
      preco REAL
  );
    CREATE TABLE IF NOT EXISTS clientes (
      id SERIAL PRIMARY KEY,
      nome TEXT,
      telefone TEXT,
      horario TEXT,
      idservico INTEGER
    );
    CREATE TABLE IF NOT EXISTS faturamento (
      id SERIAL PRIMARY KEY,
      nome TEXT,
      valor REAL,
      data INTEGER
    );
